33 public interface MetadataService {
34     /**
35      * Adds new column to table, which can be either empty or can have data already. When existing columns
36      * already have data this function will create ".top" file in addition to column files. ".top" file contains
37      * size of partition at the moment of column creation. It must be used to accurately position inside new
38      * column when either appending or reading.
39      *
40      * <b>Failures</b>
41      * Adding new column can fail in many situations. None of the failures affect integrity of data that is already in
42      * the table but can leave instance of TableWriter in inconsistent state. When this happens function will throw CairoError.
43      * Calling code must close TableWriter instance and open another when problems are rectified. Those problems would be
44      * either with disk or memory or both.
45      * <p>
46      * Whenever function throws CairoException application code can continue using TableWriter instance and may attempt to
47      * add columns again.
48      *
49      * <b>Transactions</b>
50      * <p>
51      * Pending transaction will be committed before function attempts to add column. Even when function is unsuccessful it may
52      * still have committed transaction.
53      *
54      * @param name                    of column either ASCII or UTF8 encoded.
55      * @param symbolCapacity          when column type is SYMBOL this parameter specifies approximate capacity for symbol map.
56      *                                It should be equal to number of unique symbol values stored in the table and getting this
57      *                                value badly wrong will cause performance degradation. Must be power of 2
58      * @param symbolCacheFlag         when set to true, symbol values will be cached on Java heap.
59      * @param type                    {@link ColumnType}
60      * @param isIndexed               configures column to be indexed or not
61      * @param indexValueBlockCapacity approximation of number of rows for single index key, must be power of 2
62      * @param isSequential            for columns that contain sequential values query optimiser can make assumptions on range searches (future feature)
63      */

66             int type,
67             int symbolCapacity,
