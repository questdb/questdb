   **`hasNext()` rewrite — state machine for keyed emission:**

   Replace the current simple loop with a state machine. The cursor must
   emit one row per `hasNext()` call. States:

   ```
   FETCH_DATA   — try to get next data row for current bucket
   EMIT_DATA    — about to return a data row (return true)
   EMIT_FILLS   — iterating keysMap for absent keys in current bucket
   ADVANCE      — done with bucket, advance timestamp
   ```

   The core logic (pseudocode):

   ```java
   @Override
   public boolean hasNext() {
       if (!isInitialized) {
           initialize();
           isInitialized = true;
       }

       // If we're in the middle of emitting fill rows for absent keys
       if (isEmittingFills) {
           return emitNextFillRow();
       }

       while (nextBucketTimestamp < maxTimestamp) {
           // Fetch data
           long dataTs;
           if (hasPendingRow) {
               dataTs = pendingTs;
           } else if (!baseCursorExhausted && baseCursor.hasNext()) {
               dataTs = baseRecord.getTimestamp(timestampIndex);
               hasPendingRow = true;
               pendingTs = dataTs;
           } else {
               baseCursorExhausted = true;
               dataTs = Long.MAX_VALUE;
           }

           if (baseCursorExhausted && !hasExplicitTo) {
               // Emit fills for remaining absent keys in current bucket
               // before returning false, if we have data for this bucket
               if (hasDataForCurrentBucket && keysMap != null) {
                   isEmittingFills = true;
                   keysMapCursor.toTop();
                   fillKeyIdx = 0;
                   return emitNextFillRow();
               }
               return false;
           }

           if (dataTs == nextBucketTimestamp) {
               // Data row at expected bucket
               hasPendingRow = false;
               fillRecord.isGapFilling = false;
               hasDataForCurrentBucket = true;
               if (keysMap != null) {
                   // Find this key in keysMap, mark present, update prev
                   MapKey mapKey = keysMap.withKey();
                   keySink.copy(baseRecord, mapKey);
                   MapValue value = mapKey.findValue();
                   int keyIdx = (int) value.getLong(KEY_INDEX_SLOT);
                   keyPresent[keyIdx] = true;
                   updatePerKeyPrev(value, baseRecord);
               } else {
                   // Non-keyed: single key always present
                   keyPresent[0] = true;
                   if (hasPrevFill) savePrevNonKeyed(baseRecord);
               }
               // Check if next pending row is also in this bucket
               // (multiple keys can have data in the same bucket)
               // Do NOT advance bucket yet — there may be more data rows
               // at this timestamp.
               // Peek next row to decide:
               if (!baseCursorExhausted && baseCursor.hasNext()) {
                   long nextTs = baseRecord.getTimestamp(timestampIndex);
                   if (nextTs == nextBucketTimestamp) {
                       hasPendingRow = true;
                       pendingTs = nextTs;
                       // Still in same bucket — return this data row,
                       // next hasNext() call will handle the next data row
                       return true;
                   } else {
                       // Next row is in a different bucket. Save it as pending.
                       hasPendingRow = true;
                       pendingTs = nextTs;
                       // Before advancing bucket: emit fill rows for absent keys
                       if (keysMap != null && keyCount > 0) {
                           // Return this data row first. On next call,
                           // we'll emit fills.
                           // Wait — we need to emit THIS data row, then fills,
                           // then advance bucket. Set a flag.
                           // Actually: return this data row now. On next call,
                           // since we haven't advanced nextBucketTimestamp yet
                           // and the pending row's ts != nextBucketTimestamp,
                           // we'll fall through to the gap branch which will
                           // emit fills before advancing.
                       }
                       return true;
                   }
               } else {
                   baseCursorExhausted = true;
                   // Last data row in base cursor. Return it. Next call
                   // will see baseCursorExhausted and emit fills / stop.
                   return true;
               }
           }

           if (dataTs > nextBucketTimestamp) {
               // Gap bucket — no data rows at this timestamp.
               // For keyed: emit fill rows for ALL keys (none present).
               // For non-keyed: emit single fill row.
               if (keysMap != null && keyCount > 0) {
                   // First, emit any remaining fills for the PREVIOUS bucket
                   // if we had partial data. If hasDataForCurrentBucket is
                   // true, we need to emit fills for absent keys in the
                   // previous bucket before handling this gap bucket.
                   // Actually, with the peek-ahead above, by this point
                   // hasDataForCurrentBucket relates to the CURRENT bucket.
                   // If the current bucket had some data rows, fills for
                   // absent keys haven't been emitted yet.
                   if (hasDataForCurrentBucket) {
                       // Emit fills for absent keys in current bucket
                       isEmittingFills = true;
                       keysMapCursor.toTop();
                       fillKeyIdx = 0;
                       return emitNextFillRow();
                   }
                   // This bucket has NO data at all — emit fills for all keys
                   isEmittingFills = true;
                   keysMapCursor.toTop();
                   fillKeyIdx = 0;
                   Arrays.fill(keyPresent, 0, (int) keyCount, false);
                   return emitNextFillRow();
               }
               // Non-keyed gap
               fillRecord.isGapFilling = true;
               fillTimestampFunc.value = nextBucketTimestamp;
               nextBucketTimestamp = timestampSampler.nextTimestamp(nextBucketTimestamp);
               hasDataForCurrentBucket = false;
               return true;
           }

           if (dataTs < nextBucketTimestamp && hasPendingRow) {
               // DST fall-back: emit data row as-is
               hasPendingRow = false;
               fillRecord.isGapFilling = false;
               return true;
           }
       }
       return false;
   }
   ```

   **IMPORTANT DESIGN NOTE:** The pseudocode above is a starting point. The exact
   state machine is complex and the executor should carefully design it. The key
   constraints are:

   - `hasNext()` returns one row at a time and must be re-entrant.
   - For each time bucket with keyed queries:
     1. Emit all data rows at this timestamp (consecutive in sorted input).
     2. After all data rows, emit fill rows for each absent key (iterate
        keysMapCursor, skip keys where `keyPresent[keyIndex]` is true).
     3. Reset `keyPresent` and advance `nextBucketTimestamp`.
   - For gap buckets (no data rows at all): emit fill rows for ALL keys.
   - **DO NOT call `baseCursor.hasNext()` as a peek-ahead that corrupts
     the current record.** Use `hasPendingRow` / `pendingTs` to manage
     the one-row lookahead. The one exception: AFTER emitting a data row
     (the caller has already consumed it via `return true`), the NEXT
     `hasNext()` call can safely call `baseCursor.hasNext()` because
     the previous row is no longer needed.
   - When emitting fills for absent keys, iterate the `keysMapCursor`
     (insertion order). For each key, check `keyPresent[keyIndex]`. If
     NOT present:
       - Set `fillRecord.isGapFilling = true`
       - Set `fillTimestampFunc.value = nextBucketTimestamp`
       - Position the keysMapRecord at this key (the cursor iteration
         naturally does this)
       - Return true
     After all keys checked, reset `keyPresent`, advance bucket, set
     `isEmittingFills = false`, `hasDataForCurrentBucket = false`.
   - The `emitNextFillRow()` helper method iterates through the remaining
     keys in the keysMapCursor and emits the next absent key's fill row.
     When all keys have been checked, it resets presence, advances the
     bucket, and falls through to the main loop for the next bucket.

   **`emitNextFillRow()` method:**
   ```java
   private boolean emitNextFillRow() {
       while (keysMapCursor.hasNext()) {
           MapValue value = keysMapRecord.getValue();
           int keyIdx = (int) value.getLong(KEY_INDEX_SLOT);
           if (!keyPresent[keyIdx]) {
               fillRecord.isGapFilling = true;
               fillTimestampFunc.value = nextBucketTimestamp;
               fillKeyIdx = keyIdx;
               return true;
           }
       }
       // All keys checked — done with this bucket's fills
       Arrays.fill(keyPresent, 0, (int) keyCount, false);
       nextBucketTimestamp = timestampSampler.nextTimestamp(nextBucketTimestamp);
       hasDataForCurrentBucket = false;
       isEmittingFills = false;
       // Continue with the main loop
       return hasNext();
   }
   ```

   **Per-key prev update — `updatePerKeyPrev(MapValue, Record)`:**
   ```java
   private void updatePerKeyPrev(MapValue value, Record record) {
       value.putLong(HAS_PREV_SLOT, 1L);
       for (int col = 0; col < columnCount; col++) {
           int aggSlot = outputColToAggSlot[col];
           if (aggSlot >= 0) {
               value.putLong(PREV_START_SLOT + aggSlot,
                   readColumnAsLongBits(record, col, columnTypes[col]));
           }
       }
   }
   ```
   Where `KEY_INDEX_SLOT = 0`, `HAS_PREV_SLOT = 1`, `PREV_START_SLOT = 2`.

   The per-key prev is read from the MapValue that the keysMapCursor is
   currently positioned at. The `prevValue(col)` method must be updated:
   ```java

   }
   ```
