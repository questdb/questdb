**Fix 3 — Recursive stack overflow (line 462):**
Replace `return hasNext();` at the end of emitNextFillRow() with an iterative
loop. The refactored emitNextFillRow() should:

1. After exhausting keysMapCursor for one bucket (the existing while loop), reset
   keyPresent, advance nextBucketTimestamp, set hasDataForCurrentBucket=false,
   set isEmittingFills=false.
2. Instead of `return hasNext()`, check if the next bucket still needs fills:
   - If `nextBucketTimestamp >= maxTimestamp`: return false (past the range).
   - If there is a pending data row (`hasPendingRow && pendingTs == nextBucketTimestamp`):
     return false (the next bucket has data; let hasNext() handle it).
   - If `baseCursorExhausted && !hasExplicitTo`: return false (no more data, no TO bound).
   - Otherwise (pendingTs > nextBucketTimestamp, or baseCursorExhausted with
     explicit TO, meaning the next bucket is also a gap): reset keysMapCursor
     with `keysMapCursor.toTop()`, set `isEmittingFills = true`,
     `Arrays.fill(keyPresent, 0, keyCount, false)`, and continue the outer loop
     to emit fills for all keys in this gap bucket.

Wrap the entire method body in an outer `while (true)` loop so that consecutive
empty buckets are handled iteratively. The method structure becomes:

```java
private boolean emitNextFillRow() {
    while (true) {
        // Inner loop: scan remaining keys in current bucket
        while (keysMapCursor.hasNext()) {
            MapValue value = keysMapRecord.getValue();
            int keyIdx = (int) value.getLong(KEY_INDEX_SLOT);
            if (!keyPresent[keyIdx]) {
                fillRecord.isGapFilling = true;
                fillTimestampFunc.value = nextBucketTimestamp;
                return true;
            }
        }
        // Bucket exhausted — advance
        Arrays.fill(keyPresent, 0, keyCount, false);
        nextBucketTimestamp = timestampSampler.nextTimestamp(nextBucketTimestamp);
        hasDataForCurrentBucket = false;
        isEmittingFills = false;

        // Check if next bucket also needs fills (iterative, no recursion)
        if (nextBucketTimestamp >= maxTimestamp) {
            return false;
        }
        if (hasPendingRow && pendingTs == nextBucketTimestamp) {
            return false; // next bucket has data — let hasNext() handle it
        }
        if (baseCursorExhausted && !hasExplicitTo) {
            return false;
        }
        // Next bucket is a gap — emit fills for all keys
        isEmittingFills = true;
        keysMapCursor.toTop();
    }
}
```
