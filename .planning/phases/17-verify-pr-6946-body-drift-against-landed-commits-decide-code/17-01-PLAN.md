<objective>
Close the two safety-critical bugs surfaced by the second `/review-pr 6946` pass that did not make the first cleanup round:

1. **M-new (D-07..D-10):** `SampleByFillRecordCursorFactory.initialize()`'s keyed pass-1 key-discovery loop at ~:603 has no `circuitBreaker.statefulThrowExceptionIfTripped()` call. Master's cursor-path equivalents (`SampleByFillPrevRecordCursor:171`, `SampleByFillValueRecordCursor:183`) poll the CB as the first statement inside every `while (baseCursor.hasNext())` body. Without this poll, a long-running keyed SAMPLE BY FILL with a large base cursor can ignore cancellation / timeout during key discovery before emission starts. Commit `c1deb9b14d` fixed two emission-side sites but missed pass-1. Regression vs master.

2. **M-unit (D-26..D-27):** `SqlCodeGenerator.generateFill`'s cross-column FILL(PREV) type-compatibility check at :3605-3611 uses tag equality for types outside DECIMAL/GEOHASH/ARRAY. `ColumnType.tagOf(t)` returns `t & 0xFF`, so TIMESTAMP_MICRO (tag 8) and TIMESTAMP_NANO ((1<<18)|8) collapse to the same tag; same for INTERVAL_TIMESTAMP_MICRO vs _NANO. A MICRO-sourced cross-column PREV populating a NANO target (or vice versa) passes the compatibility check, and `FillRecord.getTimestamp` / `getInterval` copy the raw long without unit conversion — silent 1000x drift. Net-new bug in the PR.

Purpose: ship both fixes with paired regression tests following the Phase 15 D-02 same-commit rule (test restoration/addition lands in the same commit as the codegen fix, because a reverted fix must make the test fail).
