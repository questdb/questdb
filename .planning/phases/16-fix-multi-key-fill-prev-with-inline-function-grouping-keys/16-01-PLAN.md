<objective>
Close the latent correctness gap in `SqlCodeGenerator.generateFill` where non-aggregate FUNCTION and OPERATION grouping keys (`interval(lo, hi)`, `concat(a, b)`, `cast(x AS STRING)`, `a || b`) fall through into the aggregate arm of the `bottomUpCols` classifier loop. Pre-fix they are mapped to FILL_PREV_SELF and silently dropped from `keyColIndices`, so multi-key SAMPLE BY FILL produces a non-cartesian passthrough (3 rows instead of 6 for the empirical probe). Land D-02 (widened classifier predicate + alias assert) together with D-05 (aggregate-arm lock-in assertion) and 5 regression tests in a single commit, per Phase 14/15 file-clustered convention.

Purpose: Restore cartesian fill-row semantics for function-valued grouping keys; lock the classifier invariant under `-ea` so future AST-shape drift fails CI immediately.

Output:
- `SqlCodeGenerator.java` classifier widened to 4 branches (~10 line diff + 1 line aggregate-arm assert).
- 5 new `@Test` methods in `SampleByFillTest.java` (~200 line diff), alphabetically placed.
- One commit. Title <= 50 chars, long-form body describing the classifier bug + fix + test coverage.
</objective>
