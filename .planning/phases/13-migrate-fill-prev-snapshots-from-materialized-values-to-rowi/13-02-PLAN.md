<objective>
Rewrite `SampleByFillRecordCursorFactory` to store PREV snapshots as chain rowIds (one per key) instead of per-type materialized values, and shrink the `generateFill()` MapValue schema to a fixed 3-LONG layout. Per D-01, this is the required mechanism — value buffering is permanently ruled out.

Purpose: Core architectural work of phase 13. A single `long` rowId per key replaces the entire per-type dispatch matrix (KIND_LONG_BITS, KIND_SYMBOL, KIND_LONG128, KIND_DECIMAL128, KIND_LONG256, KIND_DECIMAL256, KIND_STRING, KIND_VARCHAR, KIND_BIN, KIND_ARRAY). Gap-fill emit positions `prevRecord` via `baseCursor.recordAt(prevRecord, rowId)`, and all typed getters read uniformly from chain memory. Unlocks every type currently routed through retro-fallback (Plan 04 deletes the retro-fallback machinery next).

Output: Commit 2 of the phase-13 six-commit sequence (D-07). Two production files modified. All existing tests still pass (new per-type tests land in Plan 03). Retro-fallback machinery remains temporarily — Plan 04 deletes it.
</objective>

Canonical emit pattern (PI-02 per RESEARCH.md §2.4):
```java
// In emitNextFillRow or the fill branch of hasNext, before returning true:
if (hasKeyPrev()) {  // keyed

    baseCursor.recordAt(prevRecord, prevRowId);
}
// OR (non-keyed):
if (hasSimplePrev) {
    baseCursor.recordAt(prevRecord, simplePrevRowId);
}
// Then FillRecord.getXxx(col) delegates to prevRecord.getXxx(col) without re-calling recordAt.
```
