```java
    private final RecordCursorFactory base;           // <-- `final` needs removal
    private final SortedRecordCursor cursor;

    private final ListColumnFilter sortColumnFilter;

    public SortedRecordCursorFactory(
            @NotNull CairoConfiguration configuration,
            @NotNull RecordMetadata metadata,
            @NotNull RecordCursorFactory base,
            @NotNull RecordSink recordSink,
            @NotNull RecordComparator comparator,
            @NotNull ListColumnFilter sortColumnFilter
    ) {
        super(metadata);
        this.base = base;                             // <-- LINE 56: outside try, sets caller-owned ref before it's owned
        this.sortColumnFilter = sortColumnFilter;
        RecordTreeChain chain = null;
        try {
            chain = new RecordTreeChain(              // <-- LINE 60: may throw LimitOverflowException
                    metadata, recordSink, comparator,
                    configuration.getSqlSortKeyPageSize(),
                    configuration.getSqlSortKeyMaxPages(),
                    configuration.getSqlSortValuePageSize(),
                    configuration.getSqlSortValueMaxPages()
            );
            this.cursor = new SortedRecordCursor(chain, comparator, SortKeyEncoder.createRankMaps(metadata, sortColumnFilter));
        } catch (Throwable th) {
            Misc.free(chain);                         // correctly frees partial chain
            close();                                  // <-- LINE 72: close() → _close() → Misc.free(base) — DOUBLE FREE
            throw th;
        }
    }
    // ...
    @Override
    protected void _close() {
        Misc.free(base);                              // LINE 136: null-safe; problem is base still points at caller's ref
        Misc.free(cursor);
    }
}
```

**The bug:** `this.base` is assigned on line 56 BEFORE the try block. If `new RecordTreeChain(...)` throws on line 60, the catch at line 70 calls `close()` (line 72), which cascades to `_close()`, which calls `Misc.free(base)`. The caller still owns `base` and will free it again via its own error handling. Free-twice on the base factory.

**The fix (per D-10 and RESEARCH section 6):** Remove `final` from `private RecordCursorFactory base`, move `this.base = base;` inside the try block AFTER the RecordTreeChain construction succeeds (or at least after all the throwing allocations inside try), null `this.base` in catch BEFORE calling `close()`. `_close()` stays null-safe via `Misc.free(null)` no-op.
