**Test 5 — `testFillPrevDecimalZeroVsNull` (D-17)**

```java
@Test
public void testFillPrevDecimalZeroVsNull() throws Exception {
    assertMemoryLeak(() -> {
        execute("""
                CREATE TABLE t (
                    ts TIMESTAMP,
                    d DECIMAL(10, 2)
                ) TIMESTAMP(ts) PARTITION BY DAY""");
        // Insert a legitimate 0.00 at 01:00 and a data value at 03:00.
        // Query spans 00:00..04:00 via FROM/TO so bucket at 00:00 predates any
        // data (produces NULL fill), bucket at 02:00 is between data rows
        // (produces PREV fill carrying the prior 0.00), bucket at 03:00 is a
        // data row (12.34), bucket at 04:00 carries the prior 12.34.
        execute("""
                INSERT INTO t VALUES
                    ('2024-01-01T01:00:00.000000Z', 0.00::DECIMAL(10,2)),
                    ('2024-01-01T03:00:00.000000Z', 12.34::DECIMAL(10,2))""");
        assertQueryNoLeakCheck(
                """
                        ts\td
                        2024-01-01T00:00:00.000000Z\t
                        2024-01-01T01:00:00.000000Z\t0.00
                        2024-01-01T02:00:00.000000Z\t0.00
                        2024-01-01T03:00:00.000000Z\t12.34
                        2024-01-01T04:00:00.000000Z\t12.34
                        """,
                """
                        SELECT ts, first(d) d
                        FROM t
                        SAMPLE BY 1h FROM '2024-01-01T00:00:00.000000Z' TO '2024-01-01T05:00:00.000000Z' FILL(PREV)""",
                "ts",
                true,
                true);
    });
}
```

The NULL bucket renders as empty-after-tab; the PREV-filled 0.00 bucket renders as `0.00`. Distinct strings prove the Decimal null sentinel (`MIN_VALUE`) is preserved across the fill pipeline.

```java
@Test
public void testFillPrevOuterProjectionReorder() throws Exception {
    assertMemoryLeak(() -> {
        execute("""
                CREATE TABLE t (
                    ts TIMESTAMP,
                    k SYMBOL,
                    x DOUBLE,
                    y DOUBLE
                ) TIMESTAMP(ts) PARTITION BY DAY""");
        // Two distinct keys, with a data gap between 01:00 and 04:00 so the
        // intermediate buckets must carry forward key + a+b via FILL(PREV).
        execute("""
                INSERT INTO t VALUES
                    ('2024-01-01T00:00:00.000000Z', 'A', 1.0, 10.0),
                    ('2024-01-01T00:00:00.000000Z', 'B', 2.0, 20.0),
                    ('2024-01-01T04:00:00.000000Z', 'A', 5.0, 50.0),
                    ('2024-01-01T04:00:00.000000Z', 'B', 6.0, 60.0)""");
        // The outer projection `a + b` over the inner SAMPLE BY fires
        // propagateTopDownColumns0 in SqlOptimiser, reordering the inner
        // model's columns. Pre-fix: isKeyColumn(i, bottomUpCols, ...) receives
        // factory-indexed i but reads user-ordered bottomUpCols, misclassifying
        // columns. Post-fix: factoryColToUserFillIdx is authoritative.
        assertQueryNoLeakCheck(
                """
                        column\tk
                        11.0\tA
                        22.0\tB
                        11.0\tA
                        22.0\tB
                        11.0\tA
                        22.0\tB
                        11.0\tA
                        22.0\tB
                        55.0\tA
                        66.0\tB
                        """,
                """
                        SELECT a + b, k FROM (
                            SELECT k, sum(x) a, sum(y) b FROM t SAMPLE BY 1h FILL(PREV)
                        )""",
                null,
                true,
                true);
    });
}
```

Assert every bucket between 00:00 and 04:00 has both keys A and B with correctly carried-forward `a+b` (11.0 and 22.0 respectively). If the post-fix output order differs from what's shown, adjust the expected string to match the actual (correct) output — the test's job is to lock in the post-fix behavior, which is "key carried, aggregate self-prev'd".
