<objective>
Fix native memory leaks and zero-GC violations in Multi* window percentile functions.

Purpose: The DirectArray `result` field leaks native memory when lifecycle methods (reset/reopen/toTop) are called without freeing it. Additionally, OverWholeResultSet variants heap-allocate `new double[percentileCount]` on every `preparePass2()` call, violating QuestDB's zero-GC principle. Both bugs cause memory issues under repeated query execution (e.g., cursor reopen cycles).

Output: Two production files patched with lifecycle fixes and zero-GC compliance, plus regression tests that detect leaks via reopen cycles.
</objective>

Lifecycle contract for window functions:
- close(): free ALL owned resources. Called when function is disposed.
- reset(): free and recreate mutable state. Called on cursor reset. super.reset() calls Misc.free(map).
- reopen(): close and recreate mutable state, preserve configuration. Called on factory.getCursor() after first use.
- toTop(): reset iteration position without freeing heavy resources. Called to re-iterate.
