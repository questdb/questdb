## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| SQL parser → optimizer → codegen | User-supplied query text flows through. Malformed SAMPLE BY FILL() can reach codegen. |
| In-memory pool → QueryModel instance | QueryModel is pooled and reused. clear() must reset every field or else previous-query state bleeds into the next. |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-12-01 | Tampering / data integrity | QueryModel.clear() missing the new stashedSampleByNode reset | MEDIUM | mitigate | Task 1 mandates `stashedSampleByNode = null;` in the clear() body. Without it, a stashed node from a previous query could leak into a new compile and restore an unrelated SAMPLE BY clause. |
| T-12-02 | Information disclosure | FallbackToLegacyException carrying a user-visible message or position | LOW | accept | The exception is internal — never surfaced to users. Singleton pattern with `super(0)` and no message keeps it purely control-flow. |
| T-12-03 | Denial of Service | Alias resolution in Tier 1 gate infinite-looping on malicious cyclic aliases | LOW | accept | `model.getAliasToColumnMap().get(...)` is a single hash lookup, not recursive; cannot loop. Task 3 does at most two `.get()` calls, no traversal. |
| T-12-04 | Resource leak | Exception allocation on every fallback firing | LOW | mitigate | Singleton `INSTANCE` pattern (Task 1) — exactly one instance for the lifetime of the process. `fillInStackTrace()` override avoids stack-walking overhead. |
