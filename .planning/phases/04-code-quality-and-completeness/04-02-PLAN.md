**For single-percentile functions (PercentileDiscDouble, PercentileDiscLong, PercentileContDouble):**

```java

```
To:
```java
sink.val("percentile_disc(").val(arg).val(',').val(percentileFunc).val(')');
```

The field name for the percentile Function is `percentileFunc` in all three single-percentile classes. Verify by reading the constructor.

**For multi-percentile functions (MultiPercentileDiscDouble, MultiPercentileContDouble):**

Change from:
```java
sink.val("percentile_disc(").val(arg).val(')');
```
To:
```java
sink.val("percentile_disc(").val(arg).val(',').val(percentilesFunc).val(')');
```

Note: Multi-percentile classes use `percentilesFunc` (plural) as the field name. Verify by reading the constructor. The `percentilesFunc.toPlan(sink)` call should render the array literal as `[0.25,0.5,0.75]` per D-05. If it does not render correctly, the format may need explicit array formatting -- verify in tests.

For PercentileContDouble, the function name in the string is `percentile_cont`. For PercentileDiscDouble and PercentileDiscLong, it is `percentile_disc`. For MultiPercentileDiscDouble, it is `percentile_disc`. For MultiPercentileContDouble, it is `percentile_cont`. Read the current toPlan to confirm the function name string used.
