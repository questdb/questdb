In SqlCodeGenerator.generateFill(), at approximately line 3496 (just before the
`return new SampleByFillRecordCursorFactory(...)` call), the metadata passed to
the factory constructor is `groupByFactory.getMetadata()`. This GROUP BY metadata
has timestampIndex=-1 because GROUP BY does not track a designated timestamp.

Fix: create a metadata copy with the correct timestamp index. Replace the
metadata argument on line 3498 from `groupByFactory.getMetadata()` with a new
local variable:

```java
final GenericRecordMetadata fillMetadata = GenericRecordMetadata.copyOf(groupByFactory.getMetadata());
fillMetadata.setTimestampIndex(timestampIndex);
```

Then pass `fillMetadata` as the first argument to SampleByFillRecordCursorFactory
instead of `groupByFactory.getMetadata()`.

Note: `copyOf()` may return the same instance if the input is already a
GenericRecordMetadata. Since we need to mutate the timestamp index without
affecting the underlying GROUP BY metadata, use `GenericRecordMetadata.copyOfNew()`
instead -- this always creates a fresh copy. Check the source to confirm
`copyOfNew(RecordMetadata that)` exists (it does, at line ~125 of
GenericRecordMetadata.java).

This single change resolves 40 tests that fail with
"Timestamp column index expected:&lt;N&gt; but was:&lt;-1&gt;".
